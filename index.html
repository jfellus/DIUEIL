Versioning et forges logicielles

3

But

  • Sauvegarder l'état du code source, revenir à des versions précédentes
  • Documenter et tracer l'historique des modifications
  • Travailler à plusieurs sur un même projet

Concepts

  • Commit : Contient les modifications depuis la dernière sauvegarde (sauvegarde incrémentale)
  • Branche : liste de commits. On crée une branche à partir d'un commit, puis on commit sur cette branche qui diverge alors de la branche de départ. 
  • Merge : On peut fusionner une branche A à une branche B pour integrer les commits de A à B
  • Projet : graphe de commits (divergence = branche, convergence = merge) 
  • Resolution de conflits : intervention manuelle si commits rentrent en collision lors d'un merge

Forge logicielle 

  • Gestion des utilisateurs, droits de lecture, modification
  • Visualisation agréable du graphe du projet & des contributions
  • Bugtracking : notion d' "Issue"
  • Intégration continue (CI) : automatisation de tests et déploiements
  • Dans ce module, nous utilisons gitlab, basé sur le système de gestion de versions git

Environnements virtuels (virtualenv)

4

Isolation : Pour quoi faire ?

  • Vous n'avez pas les droits administrateur sur la machine -> impossible d'installer des logiciels
  • Vous ne voulez pas polluer votre environnement python actuel (e.g., conflits de version)
  • Vous voulez tester votre programme dans un environnement "minimal" et "sain"
  • Vous voulez verifier que votre programme fonctionne chez les autres

Comment faire ?

virtualenv monenv   # Crée un nouvel environnement dans ./
virtualenv -p /usr/bin/python2.7 monenv  # Choix de l'interpreteur

source monenv/bin/activate # Active l'environnement virtuel

pip install numpy # Installer un package localement

python monscript.py  # Lancer un script

pip freeze > requirements.txt # Sauvegarder la liste des packages

deactivate # Quitter l'environnement et revenir au système "global"

pip install -r requirements.txt # Installer la liste des packages sauvegardée

 

  • Ressource : https://virtualenv.pypa.io/en/latest/

  • Vous voulez tester votre programme dans un environnement "minimal" et "sain"

API : Application Programing Interface


6

Pour quoi faire ?

  • Couche d'abstraction : point d'entrée indépendant de l'implémentation spécifique (machine/langage/algo)
  • Interoperabilité : permettre à d'autres applis d'appeler les fonctionnalités de mon appli
  • Standardiser les fonctionnalités d'une famille d'applications ayant un but similaire
  • Exemple phare : navigateurs web (implémentations très différentes, même ensemble de fonctions) -> API DOM (Document Object Model)    -> on en reparle demain ! (module web)

Comment faire ?

  • Spécification : prototype, pré-conditions, comportement, post-conditions
  • Documentation
  • Contraintes : compatibilité systèmes exploitations, dépendances librairies, complexité de calcul, empreinte mémoire
